# Private Project Boundary

## Summary

The public demo is linked to the original product idea, but it must not inherit private implementation detail.

## Boundary statement

The private Device Inspector project remains the product workspace. This public repository is the interview-safe engineering abstraction.

## Allowed transfer

- Problem framing.
- Architecture patterns.
- Public-safe synthetic schemas.
- Verification and trust concepts.
- Interview-safe narrative.

## Disallowed transfer

- Private source history.
- Real inspection flows.
- Real device records.
- Production integrations.
- Local file paths.
- Private screenshots.
- Private build artifacts.

## Maintenance rule

When the private product evolves, update this public demo only at the abstraction level: docs, synthetic examples, tests, and public-safe architecture. Do not copy production implementation into this repository.
