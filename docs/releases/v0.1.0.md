# v0.1.0 — Public-safe diagnostics demo baseline

This release note is prepared for the first public baseline release of `device-inspector-demo`.

## Release scope

Included baseline surfaces:

- public-safe hardware diagnostics workflow README;
- synthetic-only demo boundary;
- architecture, validation, known-limitation, and inspection-matrix docs;
- backend Flask API, SQLite persistence, report signing, and tests;
- SwiftUI and Kotlin client skeletons;
- public security policy and contribution guide;
- pull request template;
- existing CI and public-safety checks.

## Validation before publishing

Run from the repository root when the local environment is available:

```bash
cd backend/flask_api
pytest -q
cd ../..
bash scripts/blackbox_api_test.sh
python3 scripts/check_public_safety.py
git diff --check
git status --short
```

Do not claim these commands passed unless their exact output is captured.

## Publishing path

After local `main` is synced to the validated remote HEAD:

```bash
git tag -a v0.1.0 -m "v0.1.0 — Public-safe diagnostics demo baseline"
git push origin v0.1.0
```

Manual release fallback:

```bash
gh release create v0.1.0 \
  --repo EthanSangSSS/device-inspector-demo \
  --title "v0.1.0 — Public-safe diagnostics demo baseline" \
  --notes-file docs/releases/v0.1.0.md \
  --verify-tag
```

## Honesty boundary

This release does not claim production adoption, external users, real-device validation, proprietary diagnostic access, or production AI triage. It is a public-safe demo baseline using synthetic records and deterministic logic.

## Known limitations

- Not a production diagnostic service.
- Does not process real device-owner data.
- Does not include real serials, vendor diagnostics, real defect images, or private model weights.
- Human review and privacy/security/model-risk review would be required before production use.
